GST & E-Way Login Tool Privacy Policy

Local Excel data and licence verification handling.

Last updated: 20 July 2026

This policy explains how the GST & E-Way Login Tool Chrome extension processes local Excel records, licence information and portal login data.

1. Information processed

  • Local Excel records selected by the user, which may include firm name, GSTIN, GST or E-Way Bill user ID, password, contact and remarks.
  • Registration and licence information, including mobile number, Gmail address, licence key, a randomly generated Chrome installation identifier, activation token and licence status.
  • Local import details and preferences, such as the selected file name, import time, record counts and panel settings.

2. Local Excel processing

The user manually selects an Excel file from the device. The extension reads and parses that file within the Chrome extension environment.

The Excel file and its GST or E-Way Bill credential records are not uploaded to Google Sheets, Google Drive, Samriddhi Associates or the licence server. The extension does not request Google Sign-In, Google Sheets API access or Google Drive access.

3. How local records are used and stored

  • Imported records are stored in Chrome extension local storage so the user can search them and use autofill without importing the Excel file on every use.
  • Only the record selected by the user is filled into the login fields on the official GST or E-Way Bill portal.
  • The extension does not collect browsing history and does not read or modify unrelated websites.
  • A random installation identifier may also be stored using Chrome Sync, when available, to support licence binding and reinstall activation. It does not contain GST or E-Way Bill credentials.

4. Licence verification

For registration, activation and licence verification, the extension sends the mobile number, Gmail address, licence key, random installation identifier, activation token and verification status through HTTPS to Google Apps Script endpoints operated by Samriddhi Associates.

Excel records, GSTINs, portal user IDs and portal passwords are not included in licence verification requests.

5. Data sharing

  • User data is not sold, rented or used for advertising.
  • Licence information is processed through Google Apps Script and a central licence sheet controlled by Samriddhi Associates only for registration, activation, security, support and licence management.
  • When the user proceeds with a login, the selected credentials may be submitted by the browser to the official GST or E-Way Bill portal. Those portals process the data under their own policies.
  • Data is not shared with unrelated third parties.

6. Chrome permissions

  • Storage: saves imported records, licence information, import details and user preferences in the extension environment.
  • Tabs and scripting: opens extension pages, communicates with supported portal tabs and displays or restores the autofill interface.
  • Host access: limited to the official GST portal, the official E-Way Bill portal and Google Apps Script endpoints used for licence verification.
  • The extension does not request Google identity, Gmail, Google Sheets or Google Drive permissions.

7. Retention and deletion

Imported Excel records remain in Chrome extension storage until the user clears local data or removes the extension. The original Excel file remains under the user’s control on the device.

Licence records are retained for the duration of the licence and as reasonably necessary for activation, fraud prevention, support, security and business record-keeping. Users may contact Samriddhi Associates to request correction or deletion, subject to applicable legal and contractual requirements.

8. Security

Licence requests are transmitted over HTTPS. GST and E-Way Bill credentials remain in the user’s Chrome extension storage and are used only for the user-selected autofill action and subsequent portal login.

Because the extension handles authentication information, users should protect their computer, Chrome profile and Excel file, avoid shared or public devices, and remove local data before transferring or disposing of a device.

9. User choices

  • Choose whether and when to import an Excel file.
  • Clear imported records from the extension.
  • Remove the extension to delete its local Chrome storage.
  • Contact Samriddhi Associates regarding licence information.

10. Changes to this policy

This policy may be updated when the extension’s features or data practices change. The revised date will be shown on this page.
