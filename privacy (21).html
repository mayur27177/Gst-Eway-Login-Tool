Gmail Sender Privacy Policy

Google user data handling for Samriddhi Gmail Sender.

Last updated: 16 July 2026

1. Information used

  • Configured Gmail address
  • OAuth tokens issued after consent
  • Recipient details in the local Excel file
  • User-selected attachments
  • Local send status

2. Gmail permission

The app requests https://www.googleapis.com/auth/gmail.send only to send emails prepared and initiated by the user.

3. Gmail data not requested

  • No inbox-reading permission
  • No Gmail contacts permission
  • No deletion or modification of existing messages
  • No Google Drive or Sheets permission

4. Local storage

The Gmail address, OAuth token and recipient Excel file are stored locally. The app does not operate a Samriddhi Associates server for collecting Gmail message data.

5. Sharing

Google user data is not sold, used for advertising or shared with unrelated third parties. Messages and attachments are transmitted through Gmail to user-selected recipients.

6. Google API Limited Use

Use of information received from Google APIs complies with the Google API Services User Data Policy, including its Limited Use requirements.

7. User controls

Users may disconnect the account, revoke access in Google Account settings, and delete local application files and data.

8. Security

The app uses Google OAuth and does not ask the user to enter a Gmail password into the software.
